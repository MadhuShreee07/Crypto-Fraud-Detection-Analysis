\# Crypto Fraud Detection – Key Insights



\## Dataset Overview



\* Total transactions analyzed: \*\*1000\*\*

\* Fraudulent transactions detected: \*\*108\*\*

\* Overall fraud rate: \*\*10.8%\*\*



\---



\## Key Findings



\### 1. High-Value Transactions



Several transactions exceeded \*\*150 units\*\*, indicating potentially suspicious high-value transfers that may require monitoring.



\### 2. Geographic Transaction Activity



The highest number of transactions originated from:



\* \*\*China – 184 transactions\*\*

\* \*\*Indonesia – 111 transactions\*\*

\* \*\*Russia – 58 transactions\*\*



These locations represent the most active regions in the dataset.



\### 3. Fraud Distribution by Location



Fraudulent transactions were most frequently observed in:



\* \*\*Indonesia – 19 flagged transactions\*\*

\* \*\*China – 11 flagged transactions\*\*

\* \*\*Russia – 10 flagged transactions\*\*



This suggests possible regional fraud concentration.



\### 4. Fraud Rate by Location



Some regions show high fraud ratios relative to their transaction volume:



\* \*\*Malaysia – 60% fraud rate\*\*

\* \*\*Mexico – 45.45% fraud rate\*\*



However, certain locations with \*\*100% fraud rate\*\* had very small transaction counts.



\### 5. Gas Fee Patterns



Some transactions displayed unusually high gas fees (close to \*\*0.002\*\*), which may indicate attempts to prioritize transaction processing on the network.



\### 6. Transaction Amount Patterns



Average transaction values vary across locations, highlighting differences in transaction behavior across regions.



\---



\## Conclusion



The analysis reveals patterns in transaction volume, fraud distribution, and gas fee behavior. While the dataset is simulated, the queries demonstrate how SQL can be used to detect suspicious patterns in cryptocurrency transactions.
